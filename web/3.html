JavaScript

웹 사이트 클라이언트 쪽 사용

JavaScript는 웹의 지배적인 클라이언트 측 스크립팅 언어이며 모든 웹 사이트의 99%가 이 목적으로 사용합니다. [10] 스크립트는 HTML 문서에 포함되거나 HTML 문서에서 포함되며 DOM과 상호 작용합니다. 모든 주요 웹 브라우저에는 사용자 장치에서 코드를 실행하는 JavaScript 엔진이 내장되어 있습니다. 스크립팅된 동작의 예 페이지를 다시로드하지 않고 Ajax 또는 WebSocket을 통해 새 웹 페이지 콘텐츠를로드합니다. 예를 들어, 소셜 미디어 사용자는 현재 페이지를 떠나지 않고도 메시지를 보내고 받을 수 있습니다. 웹 페이지 애니메이션(예: 개체 페이드 인 및 페이드 아웃, 크기 조정 및 이동) 브라우저 게임하기. 스트리밍 미디어의 재생을 제어합니다. 팝업 광고 또는 경고 상자 생성. 데이터가 웹 서버로 전송되기 전에 웹 양식의 입력 값에 대한 유효성 검사를 수행합니다. 사용자의 행동에 대한 데이터를 기록한 다음 서버로 보냅니다. 웹사이트 소유자는 이 데이터를 분석, 광고 추적 및 개인화에 사용할 수 있습니다. 사용자를 다른 페이지로 리디렉션합니다. 스토리지 또는 IndexedDB 표준을 통해 사용자 장치에 데이터를 저장하고 검색합니다. 라이브러리 및 프레임워크 웹 사이트의 80% 이상이 클라이언트 측 스크립팅의 일부로 타사 JavaScript 라이브러리 또는 웹 프레임워크를 사용합니다. [39] jQuery는 지금까지 가장 많이 사용됩니다. 다른 주목할만한 것들로는 Angular, Bootstrap, Lodash, Modernizr, React, Underscore 및 Vue가 있습니다. jQuery 및 Bootstrap과 같은 여러 옵션을 함께 사용할 수 있습니다. 그러나 "Vanilla JS"라는 용어는 라이브러리나 프레임워크를 전혀 사용하지 않고 대신 표준 JavaScript 기능에 전적으로 의존하는 웹사이트를 위해 만들어졌습니다.
